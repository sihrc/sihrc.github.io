Heuristic Evaluation Response

This page describes our responses to the Heuristic Evaluation review that our team mates completed on the first version of our myDish app.

Heuristic Evaluation Feedback and Responses
Heuristic Feedback Severity Rating Response
The addEntry button on the homepage caused me to pause. I was not sure if an “entry” was a dish. Maybe rename to Add Dish or something? 2 Change "Add Entry" to "Add Dish". "Add Dish" specifically indicates dishes are being added, while Add Entry is a general statement in comparison.
Creating a dish with no name displays strangely and there is no validation on the form to make sure a name was entered 3 Fix error so that Dish can be added without title. It is possible users wouldn't feel like adding a title till later (especially if they're in a rush or tired) and come across this error.
When creating a new dish, it’s not obvious what the drop­down menu does until you click on it 3 Put a placeholder of “Categories” on the Add Entry page. This will notify users that selecting the "Categories" button will pull down categories.
When I was editing a recipe, I cleared the data in the recipe and the notes column. There was no label or placeholder text in these inputs and I forgot what was supposed to go in each input. The user will be confused without labels if they make a mistake which could mean they enter a recipe in the notes column or vice versa. 4 Label the fields on the Add/Edit Entry pages.This way users will know what to enter into the fields in case they delete all the contents.
Creating a dish with the same name as an existing dish doesn’t display properly­ it results in two identical dishes. There is also nothing to warn me that I may have that dish already. 3 Fix error so that Dishes can be added with a title of an existing dish. We expect user wouldn't mind having two dishes of the same name, but would mind getting error messages indicating similar titles.
Nothing reminds me to save if I try to hit “back” after editing a recipe, thus losing my changes. 3 Add popup on Add/Edit Entry page to make sure people want to cancel, when they press the cancel/back button (the buttons on the bottom of the page). The popup will not appear if user selects back arrow (located at top of pages). The popups would remind user to save in case changes were made. In case where user selects back arrow, we decided users wouldn't want reminder, especially if they are just viewing the entries in the Edit page (The popups would just be irritating).
I don’t understand the point of having the entree/appetizer/ etc. categorization. It doesn’t seem to do anything, and I would assume that if you have the recipe open, you would know what kind of dish it was anyway. 2 Leave the categorization option. The original purpose was to allow users to search for Dishes based on category, but we were unable to implement the search option due to time constraints.
The “...” on “Other...” on the Create a Dish suggests to me a text input to enter my own preferences instead of selecting the string “Other...”. 2 Take out the "Other..." category and "Bread" category examples. This will be less confusing to users during user testing.
There is no rhyme or reason to how the recipes are ordered, making it hard to locate particular recipes, especially when there are a lot of them. 3 Make no change. We think it would be obvious to our users that the dishes are in reverse chronological order. (Our user testing showed users didn't really care about the date, so long as they knew the order they cooked the meals in)
No way to delete recipes 3 Make no change. Our users are unlikely to delete recipes (since meals that they would grow tired of would sit at the very bottom of the main page, and they wouldn't bother adding meals that they don't like).
“back” and “cancel” are essentially the same 0 Change the "Back" button at the bottom of the Edit Entry page to "Cancel" in order to be consistent with the "Cancel" button on the Add Entry page (they essentially do the same thing which is to go back to the main page)
When editing a dish, pushing the retake photo button brings me to the homepage. 2 Fix error so that the "retake photo button" retakes a photo instead of going to the homepage. The retake photo button should be acting like a retake photo button
On the addEntry page there is a button labeled Retake Photo. This is a little counterintuitive because I had not previously “taken” (selected) a photo. 1 Change the "Retake Photo" button to read "Take Photo"
